# /agentstack-index-docs

Makes the project's **documentation** (markdown, plain text, ADRs, changelogs,
runbooks) searchable by the agent in future sessions, so answers to questions
like "what is our auth model?" or "how do we release?" ground in your own
docs instead of generic training data.

## Why docs only (and not source code)

- **Cursor already indexes code locally.** Uploading the codebase to a hosted
  RAG duplicates that work and pushes source into a network round-trip you
  do not need for code search.
- **Source code often contains secrets, internal logic, and proprietary
  IP.** Keep it on your machine. Docs are the intentionally-shareable layer.
- **Docs are the highest-signal-per-token surface for AI context** — READMEs,
  ADRs, architecture notes, runbooks. That is exactly where grounding pays
  off the most.

If you genuinely need semantic code search (rare — Cursor's built-in is
usually enough), do it yourself with `rag.collection_create` +
`rag.document_add`; this command deliberately refuses.

## Prerequisites

Call `buffs.get_effective_limits` to verify RAG tier:

- **Starter** — fine for most docs folders (≤ 100 chunks).
- **Pro+** — recommended; enables hybrid search and MMR.
- **Free** — not sufficient; suggest upgrade via `/agentstack-scaffold-backend`.

## Default file selection

Included by default:

- `**/*.md`, `**/*.mdx`
- `**/*.rst`, `**/*.txt`
- `README*`, `CHANGELOG*`, `ROADMAP*`, `CONTRIBUTING*`, `ADR*`, `RFC*`
- Any file under `docs/`, `documentation/`, `handbook/`

Excluded by default (always): `node_modules/`, `dist/`, `build/`, `.next/`,
`.venv/`, `__pycache__/`, `vendor/`, `*.lock`, `*.min.*`, `*.map`, anything
matching `.gitignore`.

Additional excludes may be listed in `.cursor/agentstackignore` (same syntax
as `.gitignore`). Create the file if it is missing.

Refuse to index:

- Any file whose path contains `secret`, `secrets`, `credentials`, `.env`,
  `private`, `id_rsa`, `keys/`. Surface a warning instead.
- Any file > 500 KB (split it manually first, most docs are well under this).

## Steps

1. `rag.collection_create` with `name="my-project-docs"` if it does not
   exist.
2. Walk the working tree applying the rules above.
3. For each included file:
   - Chunk at paragraph boundaries (~ 500 tokens, 50-token overlap).
   - Compute a content hash (`sha256` of the chunk body).
   - Skip chunks whose hash is already in the collection (filter by
     `metadata.hash`).
4. Batch `rag.document_add` (up to 50 docs per request) via
   `commands.execute_batch`. Set `metadata.path`, `metadata.hash`,
   `metadata.kind` (`readme` / `adr` / `runbook` / `general`).
5. Print a summary: `N files, M chunks, K skipped (unchanged), R refused`.

## Opt-in incremental updates

Set in `~/.cursor/settings.json`:

```json
{ "agentstack.autoIndexDocs": true }
```

When enabled, the `afterFileEdit` hook re-indexes only files that match the
selection rules above when you save them. Quota-aware: skips silently when
the project would exceed the tier.

## Skill hint for future sessions

After indexing, print:

> Collection `my-project-docs` is ready. To search:
>
> ```json
> { "action": "rag.search", "params": { "collection": "my-project-docs", "query": "<question>", "top_k": 5, "hybrid": true } }
> ```

The `agentstack-rag` skill already knows about this collection; no further
configuration needed.

## Pitfalls

- Indexing a very large `docs/` folder can consume significant daily token
  budget. The command shows the estimate and asks for confirmation if it
  will use more than 50 % of the remaining budget.
- Keep generated docs (API reference autogenerated from code, sitemap
  dumps) out of the index — they are low signal and churn a lot.
